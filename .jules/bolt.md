## 2025-06-29 - [Optimization of High-Frequency Animations]
**Learning:** Using React state to drive animations based on high-frequency events like `scroll` or `mousemove` triggers a full React reconciliation and re-render cycle on every frame. This creates significant CPU overhead and can lead to jank. Offloading these animations to Framer Motion hooks (`useScroll`, `useTransform`, `useMotionValue`, `useSpring`) allows updates to happen directly in the DOM/compositor without involving the React render loop.
**Action:** When implementing parallax, tilt, or any animation driven by frequent events, use `MotionValues` and Framer Motion's hook-based API instead of React state. Ensure CSS `transition` properties are removed from target elements to avoid conflicts with Framer Motion's spring-based engine.
